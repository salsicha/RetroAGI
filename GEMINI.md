
Create a set of models that play Super Mario Bros based on Predictive Coding. The models should use the pyhgf python package. The models should not need any prior training data, but should be able to learn as they play the game, with the model weights periodically being serialized and saved to disk. Make one of the models an occipital model that handles perception, and include a decoder to show that the occipital model reconstructs the input frames from the game. Make another model a temporal model that handles sequences of actions and sprite positions, that takes the latent output of the occipital model, and include a decoder that reconstructs the position of game sprites over time. Make another model a hippocampus, perhaps using a sparse distributed representation, that takes the latent output of the occipital and temporal models, that learns spatial relationships, include a decoder model that reconstructs the spatial relationships of sprites. Make another model a motor model that decides and outputs the keyboard presses that make Mario move. Also include a prefrontal model that takes latent input from the temporal and hippocampus, and outputs latent output to the motor model, and include a single normalized parameter that can be manually set that determines if the prefrontal will plan for a 'speed run' if set to zero or 'max coins' if set to one.
